# Unity 项目 Assets 目录结构说明

本项目的 Assets 目录采用标准 Unity 项目架构，便于管理和维护。

## 目录结构

### Scripts (脚本)
存放所有 C# 脚本文件，包括游戏逻辑、组件脚本等。

### Materials (材质)
存放材质文件 (.mat)，用于定义物体的外观和渲染属性。

### Prefabs (预制体)
存放预制体文件 (.prefab)，包含可重用的游戏对象模板。

### Textures (纹理)
存放纹理图片文件，用于材质的贴图，如漫反射贴图、法线贴图等。

### Models (模型)
存放 3D 模型文件 (.fbx, .obj, .blend 等)。

### Animations (动画)
存放动画文件 (.anim, .controller)，包括动画剪辑和动画控制器。

### Audio (音频)
存放音频文件 (.wav, .mp3, .ogg 等)，包括背景音乐和音效。

### PhysicsMaterials (物理材质)
存放物理材质文件 (.physicMaterial)，定义物理碰撞属性。

### Resources (资源)
存放运行时可动态加载的资源文件。

### Editor (编辑器)
存放编辑器专用脚本和工具，用于扩展 Unity 编辑器的功能。

### Plugins (插件)
存放第三方插件和原生代码库。

### Shaders (着色器)
存放自定义着色器文件 (.shader)，用于高级渲染效果。

### Fonts (字体)
存放字体文件 (.ttf, .otf)，用于 UI 和文本渲染。

### UI (用户界面)
存放 UI 相关资源，如 UI 预制体、UI 材质等。

### Sprites (精灵)
存放 2D 精灵图片，用于 2D 游戏开发。

### Scenes (场景)
存放场景文件 (.unity)，已存在。

### Settings (设置)
存放项目设置相关文件，已存在。

## 使用建议

1. 保持目录结构清晰，避免将文件放在错误的目录中
2. 使用有意义的命名规范
3. 定期整理和清理未使用的资源
4. 对于大型项目，可以在这些目录下创建子目录进行进一步分类

## 注意事项

- Unity 会自动为每个文件生成对应的 .meta 文件，请勿手动删除
- 某些特殊资源可能需要放置在特定目录中，请参考 Unity 官方文档
